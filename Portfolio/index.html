BrewView

BrewView remains a work in progress. As my unit two project for GA, the current version is a compilation of all of the breweries in CT. It is built with MongoDB, Mongoose and Express on the backend, and EJS/jQuery (as well as a bit of vanilla JS) on the front end. BrewView uses a third party API which is queried via the npm request module. It includes features such as maps, authentication, sessions, and autocomplete, as well as some CSS toys, such as the hamburger icon (on small screen sizes) and the carousel. The admin user has the ability to create, edit, and delete breweries. Typical users can view information about each brewery and then add comments if they are logged in.

Resources Used: jQuery, MongoDB, Mongoose, Express/EJS, CSS/CSS Frameworks, Node.js

Tabs on Food

Built to help my wife narrow down food intolerances, this app allows users to log meals, track patterns, and help narrow down which foods may make them sick. Built with React on the front end and Ruby on Rails and postgresql on the backend, this app is equipped with full authentication and dynamic updates. After logging a meal, users can add comments and toggle whether or not it made them sick. A graph at the top as well as emojis help the user track their food triggers. Additional features include password reset, token authentication, a sudo password reset, as well as some helpers such as React Context. This app is also built in React Native (see the next project).

Tabs on Food - With React Native

Built with React Native, this was my proof of concept for GA. I wanted to learn something new in a day and implement it over the weekend. Built on the same backend as the previous project, this is the lightweight mobile companion app. While lacking a few of the bonus features, such as inline food emojis, password reset, and password change, it has much of the same look and feel as the browser-based app. A user can log in, add meals, toggle what made them sick, and view a chart of their top food triggers. It is connected to the same database as the web version so all changes will be saved and updated.

Resources Used: React Native, Ruby on Rails, Postgresql, Knock/JWT, Node.js
