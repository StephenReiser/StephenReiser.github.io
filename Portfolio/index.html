BrewView

BrewView remains a work in progress. As is, it is a compilation of all of the breweries in CT. It is built with MongoDB, Mongoose and Express on the backend and EJS/jQuery (as well as a bit of vanilla JS) on the front end. This was my unit 2 project for GA. It utilizes a 3rd party API which is queried via the npm request module. It is filled with features such as maps, authentication, sessions, autocomplete, as well as some CSS toys such as the hamburger icon (on small screen sizes) and the carousel. The ability to create, edit, and delete breweries is available to a admin user. Typical users can view information about each brewery and then add comments if logged in

Resources Used: EJS, jQuery, MongoDB, Mongoose, Express, CSS/CSS Frameworks

Tabs on Food

Built to help my wife narrow down food allergies, this app allows a user to log meals after you ate to help track patterns in what may or may not make you sick. Built with React on the front end and Ruby on Rails and postgresql on the backend, this app comes with full authentication and dynamic updates. Users can then go back and add comments and toggle if the meal made them sick or not. A graph at the top as well as emojis help the user track their triggers. Additional features include password reset, token authentication, a sudo password reset, as well as some helpers such as React Context. This app is also built in React Native (see the next project)

Resources Used: React, Ruby on Rails, Postgresql, Knock/JWT, CSS/CSS Frameworks

Tabs on Food - With React Native

Built with React Native, this was my proof of concept for GA. I wanted to learn something new in a day and implement it over the weekend. Built on the same backend as the previous project, this is the lightweight mobile companion app. While lacking a few of the bonus features such as inline food emojis, password reset, and password change, it has much of the same look and feel as the browser based app. A user can login, add meals, toggle what made them sick, and view a chart of their top triggers. It is connected to the same database as the web version so all changes will be saved and updated.

Resources Used: React Native, Ruby on Rails, Postgresql, Knock/JWT
